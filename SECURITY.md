# Security Policy

## Reporting data integrity issues

If you find suspicious, incorrect, or potentially harmful dataset entries, please open a private security report via GitHub Security Advisories or contact repository maintainers through a private channel.

Include:

- Affected CIDR entry/entries
- Why the entry is incorrect or suspicious
- Supporting evidence (authoritative sources)
- Any urgency or impact considerations

## Response targets

Maintainers aim to:

- Acknowledge reports within 7 days
- Triage and classify severity
- Publish fixes and provenance updates as quickly as practical

## Scope

This policy covers repository data integrity and release artifacts. It does not provide operational guidance for network circumvention or bypass techniques.
